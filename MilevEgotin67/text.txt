git commands
Git е мощна система за контрол на версиите. Ето най-често използваните команди, организирани по тяхната функция: 
1. Първоначална настройка и инициализация
Тези команди се използват при стартиране на нов проект или конфигурация на вашата среда.
git config: Настройва потребителско име и имейл за вашите коммити. 0.4.9, 0.4.15
git init: Инициализира нова празна Git директория в текущата папка. 0.4.7
git clone <url>: Създава локално копие на отдалечено (remote) хранилище. 0.4.6 
2. Основен работен процес (Core Workflow) 
Команди за управление на промените във вашите файлове.
git status: Показва текущото състояние на работната директория (кои файлове са променени, кои са добавени в "staging area"). 0.4.3
git add <file>: Добавя промените от даден файл в "staging area" (подготовка за коммит). Използвайте git add ., за да добавите всички промени. 0.4.9
git commit -m "съобщение": Записва състоянието на подготвените промени в историята на проекта. 0.4.11
git diff: Показва разликите между работната директория и последното записано състояние. 0.4.7 
3. Разклонения (Branching) и Сливане (Merging) 
Използват се за работа по различни функции изолирано.
git branch: Показва списък с всички клонове. С git branch <name> създавате нов клон. 0.4.6
git checkout <branch>: Превключва към посочения клон. (В модерните версии се използва и git switch <branch>). 0.4.9
git merge <branch>: Слива промените от посочения клон в текущия. 0.4.11 
4. Работа с отдалечени хранилища (Remote) 
Команди за синхронизация с платформи като GitHub, GitLab или Bitbucket. 
git remote add origin <url>: Свързва локалното хранилище с отдалечено такова. 0.4.11
git push origin <branch>: Изпраща локалните коммити към отдалеченото хранилище. 0.4.9
git pull: Изтегля и веднага слива промените от отдалеченото хранилище в локалното. 0.4.5
git fetch: Само изтегля промените от отдалеченото хранилище, без да ги слива автоматично. 0.4.2 
5. Поправки и история
git log: Показва историята на всички коммити в текущия клон. 0.4.3
git stash: Временно "скрива" текущите незаписани промени, за да можете да работите по нещо друго. 0.4.15
git reset: Отменя промени (внимавайте с параметъра --hard, тъй като е деструктивен). 0.4.9
git revert <commit-id>: Създава нов коммит, който отменя промените от посочен предишен коммит (безопасно за публична история). 0.4.9
